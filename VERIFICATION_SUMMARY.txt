═════════════════════════════════════════════════════════════════════════════════
                             HOW ANNUAL RENEWALS WORK
═════════════════════════════════════════════════════════════════════════════════

Day 1 (Purchase):
  → User purchases annual plan
  → Stripe creates subscription with 12-month billing cycle
  → current_period_end = 12 months from now
  → Payment charged immediately

Month 11:
  → Stripe automatically collects payment for next renewal
  → No action needed from application

Day 365 (Renewal):
  → If payment successful:
    • Stripe renews subscription automatically
    • current_period_start = Previous end date
    • current_period_end = New date + 12 months
    • Webhook fires: customer.subscription.updated
    • Database updates with new period dates

  → If payment fails:
    • Subscription status = past_due
    • Stripe retries for 4+ days
    • User's plan reverts to free
    • User receives payment notification
